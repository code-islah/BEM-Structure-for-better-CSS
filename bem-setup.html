Lets Organize the CSS using BEM Method with Root Variables

Lets check out some problems first »

  • Specificity problem

    • IDs & Classes have their own values, about 100 > 10, choosing one of them as a selector in terms of higher specificity is very much important, but there may be clash between class and ID when both come together. More extra caution is needed when selecting ID, its good to go with javascript, styling a unique element and more.

  • Class name similarity

    • Styling over and over again, creating or choosing a new class name will be disturbing so this method is there to help us keep remembering of the element and finding a new name gets easier.

  • A common naming rule

    • To avoid unnecessary usage of id or class names for any element in terms of a group of people working together as a team in a document. A methodology is very much needed in this field to solve this type of problem.

BEM is here to solve those problems »

  • Solution to specificity problem

    • If we define a specific selector for all the elements which value or point is equal to all others selectors will help to avoid clash between selectors, For example: if a container has an id , then if we select an element with the id , while in the other hand classifing the same header for furthur modification, there will be clash between class and id, though id has a higher specificity so it applies immediately. Use of class selector for every element will solve the problem of specificity, the point of classied elements would be same not more than 10points without any element selector added to its front.

  • Solution to class name similarity

    • If we select a class name which has been selected before then the style will be applied as per the predefined style, so this next problem is that every element must have a separate class name to identify and style accordingly. Here BEM solves it by giving a methodology of naming by the name of block or its parent.

  • Solution to a common naming rule

    • When a formal declration is made to use BEM among a group of people, the problem of selecting each class by individual choices will disappear. The working flow will have no speed breaker infront of it, inshaAllah.

B = Block

Block defines a meaningful classified container inside a div tag, a parent or a semantic element.
For example : semantic tag 'Header' or div tag classified semanticly '.card' is a container or parent so it can be defined as a block.
Letters, digits and dashes are used for naming.
Syntax : block__element (block two underscore then element).

E = Element

Element doesnt define a meaning at all but it is related to the container, as a child of the container. Example : header__title(h2), here h2 is classified as title is an element inside the header block. For element we can also use letters, digits and dashes for naming. Syntax : block__element (header __paragraph).

M = Modifier

Modifies block and element both. Example : header__title--large, when a specific style is needed for a group of classes this comes handy. We can use same materials for modifiers which are already available to block and element, there is a little bit of change in the appearance of syntax for recognizing. Syntax : block__element > block/element--modifier, .card--large, .feedback--medium, .feedback__button--large.

Points to be noted

  • Dash entity '-' is important for joining multiple words together. !important. Example : list-item-number, cardone vs card-one, the first one interrupt the readability.
  • Inside an unorder list here comes an image tag --> ul > li > img / li / li , so here ul is the list and all the li are it's items, so the image tag is the child of the child(li) of the parent(ul), in this case it cannot be selected as list__item__img, BEM is not recommended to it, (because it stands for block then element and then modifer, block__element--modifier, cannot have anything more that these) so that we can take the item as a block and the image as its element.

CSS Variables
In the Root Selector

Variables are extremely useful for large projects, changing colors of the texts, amount of padding and margins, different values for different properties, likewise every aspect of a document in a short span of time wasnt easy, here variables have a lot of work to do, by changing only the variables will affect all the properties which have values from these variables.

How to use ?

First of all, root of a html page has to be selected to apply all the values for the variables, and that is :root (colon then root),
Two dash (--) before any property name defines a variable, --header-bg: #color; We have just created a variable for a background color, can be applied to anything requires color value, but a specific name with a specific value helps a lot even if different properties are needed to use the same value once again.
Example to that is :
--header-bg: #000;
--section-bg: #000;

Live Example :

:root {

/* || COLORS */
--header-bg: #6462B6;
--header-text: #f1f1f1;
--main-bg: purple;
--main-text: #000;
--border-color: bisque;
--list-item: steelblue;

/* || FONTS */
--ff-style: cursive;
--ff-deco: sensation;
--fs-small: 0.8rem;
--fs-large: 1.5rem;
--fw-thin: 300;

/* || PADDING & MARGIN */
--padding-avg: 0.5rem 1rem;
--margin-avg: 0.5rem 1rem;

/* || Utility */
--l-gradient: linear-gradient(to top, purple,steelblue);
--r-gradient: radial-gradient(skyblue, steelblue);
--border: 2px solid #000;
--shadow: 0 1px 1px 0 rgba(255,255,255,0.2), 0px -1px 2px 0 rgba(255,255,255,0.3);
}

/* Header Section starts */

body {
background: var(--l-gradient);
}

.header {
background: var(--r-gradient);
}

.header__title {
background: rgba(255,255,255,0.2);
font-family: var(--ff-cursive);
font-size: var(--fs-large);
padding: var(--padding-avg);
}

.header__text {
font-size: var(--fs-small);
font-weight: var(--fw-thin);
padding: var(--padding-avg);
}

/* Header section ends */

/* Main section starts */

.content {
padding: var(--padding-avg);
margin: var(--margin-avg);
}

.content__title {
padding: var(--padding-avg); box-shadow: var(--shadow);
font-weight: var(--fw-thin);
}

.content__text--large {
border: 1px dotted var(--border-color);
padding: 0.3rem;
}

/* Main section ends */
